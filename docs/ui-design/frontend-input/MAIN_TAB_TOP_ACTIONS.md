# 主标签页顶部动作（Main Tab Top Actions）

## 统一原则（Unified Rule）

- 主标签页顶部动作属于 `MainTabShell.AppTopBar`，只处理当前根 Tab 的一级入口。
- 顶部动作必须由页面显式声明；`MainTabShell` 不提供默认 `search/more`。
- 顶部动作必须使用显式动作名，例如 `search`、`more`。
- `refresh` 不放在主 Tab 顶栏；刷新属于内容区局部动作，例如发现来源状态条、RSS 订阅概览或列表区。
- 未完成的次级流程不显示顶栏占位按钮；入口必须落在页面内容区或对应二级页中。
- `more` 不是全局同一菜单；每个主 Tab 必须按该页面文档声明自己的入口边界。

## 页面规划（Page Plan）

| 页面（Page） | 顶部动作（Top Actions） | `more` 原文档状态（Original Planning Status） | 当前 demo 允许行为（Allowed Demo Behavior） | 不允许（Not Allowed） |
|---|---|---|---|---|
| 书架（Bookshelf） | `search`、`more` | `书架/文字稿.md` 明确“更多图标：打开书架级更多菜单，后续定义”。 | 可打开书架级更多菜单；当前只承载已规划书架链路入口：批量管理、分组管理、本地书导入、排序与筛选。书架显示设置仍由“我的书架”右侧齿轮进入。 | 不把书架显示设置塞进顶部更多；不打开书籍单本操作底表；不临时生成未规划菜单项。 |
| 发现（Discover） | 无 | 旧稿曾写顶部搜索/更多，但搜索入口已在内容区，更多相关流程暂缓。 | 不显示顶栏搜索/更多；搜索通过内容区 `SearchEntry` 进入书籍搜索；来源切换、分类、刷新等由内容区控件承载。 | 不生成来源选择底表、更多分类页、发现设置页或榜单详情页；不放置无闭环的占位更多按钮。 |
| RSS | 无 | 旧稿曾写顶部搜索/更多，但 RSS 搜索、订阅管理、添加订阅源、条目详情、分组管理均未形成当前闭环。 | 不显示顶栏搜索/更多；后续 RSS 搜索和订阅管理必须在页面内容区或二级页明确后再加入。 | 不生成订阅管理、添加订阅源、条目详情、分组管理菜单；不放置无闭环的占位按钮。 |
| 设置（Settings） | 无 | 旧稿曾写“搜索/更多保留入口”，但会与设置分组入口、二级设置页和破坏性操作边界冲突。 | 不显示顶栏搜索/更多；设置入口全部通过内容列表进入二级页。 | 不在设置首页直接执行导入、导出、恢复默认、清空、删除等操作；不放置无闭环的占位更多按钮。 |

## 搜索动作（Search Actions）

| 页面（Page） | `search` 当前行为（Current Behavior） |
|---|---|
| 书架（Bookshelf） | 进入书籍搜索页。 |
| 发现（Discover） | 顶栏无搜索按钮；内容区 `SearchEntry` 进入书籍搜索页。 |
| RSS | 顶栏无搜索按钮；RSS 搜索完整页暂缓，不显示占位入口。 |
| 设置（Settings） | 顶栏无搜索按钮；设置内搜索完整页暂缓，不显示占位入口。 |
